Выбор тестов для регресии - частый вопрос на собеседованиях.

Для этого есть специальный алгоритм, который позволяет быстро определить тесты для регресии:

- Первым делом в тесты для регресии определяют:
1) Тесты покрывающие безопасность;
2) Тесты проверяющие критически важные бизнес-функции;
3) Тесты покрывающие те области ПО, которые чаще всего меняются в разработке;
4) Тесты-функции с высокими вероятностями ошибок (принцип пестицида).
