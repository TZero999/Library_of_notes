API - Aplication Programming Interface
Интерфейс (способ) взаимодействия одной программы с другой (или компонентов системы друг с другом).

Другими словами:
Интерфейс прикладного программирования (Application Programming Interfaces, APIs) - это готовые конструкции языка программирования, 
позволяющие разработчику строить сложную функциональность с меньшими усилиями. 

API's "скрывают" более сложный код от программиста, обеспечивая простоту использования.

Отсюда API подразделяют на 2 типа:
1) Local API - API взаимодействия модулей одной системы;
2) Remote API - Удалённый API взаимодействия двух систем.

Когда мы говорим о Remote API - имеем ввиду SOAP или REST.
Именно SOAP и REST используются для связи веб-сервисов.

API появляется перед GUI, следовательно его можно тестировать довольно рано.

- Работа с API
    3 типа операций:
1) Вызов операции: Методы (GET/POST);
2) Входные данные: HTTP Request;
3) Выходные данные: HTTP Response.

API могут быть абсолютно разные, с совершенно разным набором функций.


- Способы вызова API
    3 прямых способа вызова API:
1) Вызов функции системой - Local API - общение компонентов между собой, редко тестируем.
2) Вызов метода другой системой - Remote API - самый распространённый случай для тестирования.
3) Вызов метода человеком - Составление запросов вручную (заполнение каких-то форм через запрос к API)
    и 1 косвенный:
4) GUI > API - элементы интерфейса, направляющие запросы к API.
