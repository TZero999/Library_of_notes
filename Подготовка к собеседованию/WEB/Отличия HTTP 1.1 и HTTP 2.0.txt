* HTTP 1.1:

1. текстовый формат:
Используется текстовый формат, из-за чего занимает намного больше места, дольше передается.

2. Сдваивания заголовков:
Если изучить различные реквесты и респонсы - там часто встречаются повторяющиеся заголовки.

3. Каждый запрос - отдельное соединение:
Каждый наш запрос (отправка html-кода, отправка css, отправка java-скрипта) проходят по
отдельному TCP соединению. Каждый запрос = отдельные соединения. Запросы идут друг за другом:
сначала получаем текстовую информацию html, потом инфо о стилях css, потом инфо по javascript.

* HTTP 2.0:

1. Бинарный формат:
Используются нули и единицы. Использование 0 и 1 уменьшает количество передаваемой информации.

2. Сокращение информации за счёт сжатия заголовков.

3. Все запросы = одно соединение:
Все запросы проходят по одному единственному TCP соединению.

4. Мультиплексирование:
Соединение одно, запросы и ответы передаются одновременно, одномоментно.
