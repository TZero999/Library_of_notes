DNS (Domain Name System, система доменных имен).
Само имя технологии позволяет понять, что DNS - это система, позволяющая 
находить нужный сайт по его доменному имени. 
Преимущества очевидны: запомнить название из букв и цифр куда проще, чем запомнить IP-адрес из цифр.

# DNS-сервер 
{
DNS-сервер — оборудование или программное обеспечение,
с помощью которого предоставляется доступ к системе доменных имен.

DNS-сервер хранит данные о соответствии конкретного IP-адреса соответствующему домену,
а также осуществляет кэширование информации в виде IP-адреса и
соответствующего ему домена других DNS-серверов.

DNS-сервера - физические устройства с установленным соответствующим ПО,
хранящие все данные в формате записей на компьютере.
}

# Принцип работы DNS простыми словами:
{
    1: Пользователь вводит в адресную строку браузера доменное имя.
    2: Преобразователь доменных имен обращается к DNS-серверу.
    3: Сервер передает IP-адрес браузеру пользователя.
    4: Браузер делает запрос на сервер по этому IP-адресу и отображает страницу ресурса.
}

# Основные характеристики DNS-технологии:
{
1: Хранение и управление данными распределенного характера.
Отдельный DNS-сервер хранит только информацию по делегированным ему доменам.

2: Кэширование данных.
Кэширование ускоряет загрузку нужной информации с сервера. 
При обращении к ресурсу (даже при запросе внутренних страниц) серверы проверяют связь домена и IP-ка.

Если запрашиваемый ресурс расположен, другой точке мира - скорость загрузки будет малой.
Например, пользователь регулярно делает запросы на ресурс, который находится в другом государстве.
Расположенный ближе всего к пользователю DNS-сервер кэширует данные о ресурсе
и при следующим запросе выдает их клиенту максимально быстро.

Источником кэширования, из которого поступают данные о сайте, являются первичные и вторичные DNS-серверы.
Для уменьшения уровня нагрузки DNS-сервер может хранить некоторое время определенное 
количество информации о других, не делегированных ему доменах.

3: Резервирование.
Изолированные логически и физически DNS-сервера хранят/обрабатывают информацию об одних и тех же узлах.
Т.О. обеспечивается доступность информации даже при сбое одного или нескольких серверов.

4: Иерархическая структура. 
База доменных имен организована по принципу иерархии:
Корневой домен расположен на верхнем уровне.
К нему примыкают домены первого уровня, к ним присоединяются домены второго уровня и т.д.
}

# DNS-зоны
{
    Под понятием «зона» в системе доменных имен подразумевается часть пространства DNS-имен,
    которая управляется либо группой серверов, либо одним сервером.
    
    Зона DNS применяется для размещения DNS-записей конкретного домена. 
    Каждая запись для домена создается внутри конкретной зоны DNS. 
    Принято разделять зоны обратного и прямого просмотра в зависимости от того, 
    какой поиск ведется внутри : Доменного имени по IP, или IP по доменному имени.
}

# Geo-DNS
{
    Geo-DNS — сервис использует дополнительные серверы для распределения трафика
    на основе местоположения запросов.
    В результате трафик к доменам оптимизируется за счет применения географической маршрутизации.

    Geo-DNS используются сайтами, находящимися в одной геолокации, но популярными в другой.
}

# DNS-атаки
{
    Спуфинг - использование системных уязвимостей DNS-ерверов для искажения хранящейся ифнормации.
    Спуфинг позволяет перенаправлять трафик с легитимных серверов на поддельные.

    DDoS-атаки - "падение" DNS-сервера несёт за собой недоступность ресурсов, стоящих за ним.
    Владельцы сайтов начинают нести материальные и репутационные потери.
}
