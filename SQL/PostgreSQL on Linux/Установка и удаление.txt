1. Установка:

- sudo apt update
Обновляем индекс пакетов.

- sudo apt install postgresql postgresql-contrib
Устанавливаем пакет PostgreSQl. Флаг 'contrib' позволяет
установить дополнительные утилиты и функции.

2. Роль суперюзера

- sudo -i -u postgres
Переходим к учётной записи postgres.

- psql
Получаем доступ к командной строке PostgreSQl.

Есть и альтернативный способ перехода к командной строке пользователя:
- sudo -u postgres psql

Для выхода из интерфейса командной строки пишем:
- \q

3. Новая роль

- sudo -u postgres createuser --interactive
Первый способ создать новую роль (напрямую из терминала)

- createuser --interactive
Второй способ создать новую роль (из командной строки PostgreSQl)

Output:
Enter name of role to add: tzero
Shall the new role be a superuser? (y/n) y

4. Новая БД

Система аутентификации Postgres использует по умолчанию
идею, что для любой роли, используемой для входа, существует
база данных с тем же именем, к которой роль может получить доступ.

Поэтому находясь в роли суперюзера postgres создать новую БД:

- sudo -u postgres createdb tzero
Первый способ создать новую БД (напрямую из терминала)

- createdb tzero
Второй способ создать новую БД (из командной строки PostgreSQl)

5. Открываем командую строку от имени нового пользователя
Чтобы выполнить аутентификацию с помощью 'ident', потребуется
пользователь Linux с тем же именем, что и имя роли 
и базы данных в Postgres.

Проверяем наличие пользователя с именем tzero (из терминала):
- id

Если такого пользователя нет - создаём его:
- sudo adduser tzero

Переключаемся на пользователя и подключаемся к БД:
- sudo -i -u tzero
- psql

Либо делаем это одной командой:
- sudo -u tzero psql

Текущего пользователя можно подключить к другой БД:
- psql -d postgres

После входа желательно проверить данные подключения (из командной строки PgSQL):
- \conninfo

Должно будет быть отображено сообщение следующего типа:
You are connected to database "tzero" as user "tzero" via socket in "/var/run/postgresql" at port "5432".
