* Что такое Swagger?
Swagger - особый инструмент, позволяющий автоматически задокументировать 
API проекта на основе его кода. Документация созданная таким образом 
облегчает понимание API для компьютеров и людей.

Т.О. на основе кода или набора правил Swagger генерирует документацию 
для API в формате JSON-файла.

* Что такое "спецификация"?
Спецификация - набор правил, описывающий как должна выглядеть или работать 
описываемая ей технология.

* OpenAPI != Swagger, но иногда...
Альтернативное название - OpenAPI. Но это не совсем корректно.
Названием "OpenAPI" на самом деле обозначают спецификацию, по которой работает
Swagger, но иногда это название применяют для обозначения самого сваггера.

Раньше сама спецификация называлась "Swagger", но в 2015 году её переименовали в
"OpenAPI" и передали другой команде разработчиков

* Что можно делать с помощью Swagger?
1) Быстро документировать API проекта, чтобы не писать документацию с нуля;
2) Использовать сгенерированный JSON-файл, чтобы внедрить сторонний API на свой проект;
3) Создавать документацию для проекта, используя JSON-файл как основу, фундамент.

* Способы создания документации:

- На основе кода:
Инструмент "парсит" API и на его основе генерирует документацию. Простой способ.
От разработчика не требуется знать спецификацию и писать что-то помимо кода.

Но им не стоит пренебрегать и использовать его только для максимально быстрого 
создания документации.

- На основе спецификации:
Используется OpenAPI (спецификация Swagger). Способ более сложный, необходимо 
знать правила, на основании которых описываются сущности кода, чтобы инструмент
понял написанное и автоматически сгенерировал документ. 

Такой подход более правильный, такая документация более понятна и читаема
человеком.
Пишут такую документацию с помощью форматов JSON или YAML, либо в специальном
инструменте "Swagger Editor".
